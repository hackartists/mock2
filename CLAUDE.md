# 정독 (Jeongdok) — 프로젝트 컨텍스트

분당·판교 학부모를 위한 **Proof-of-Engagement(PoE) 기반 교육 정보 플랫폼**의 기획·디자인 워크스페이스.

## 한 문장 정의
> "분당·판교 학부모를 위한, 진짜로 정독한 학원·입시 정보에 학원 쿠폰이 따라오는 소셜 정보 플랫폼."

---

## 디렉토리 구조

```
기획서/
├── CLAUDE.md                              ← 본 파일
├── 서비스기획서.md                          ← v1: 광범위한 보상 SNS (Top-down)
├── 서비스기획서_브레인스토밍.md              ← v2: 4단계 브레인스토밍, PoE 로컬-니치로 수렴
├── 서비스기획서_v3_PoE학부모교육.docx        ← v3: 분당·판교 × 학부모 교육으로 정조준 (현행 기준)
└── mock/
    └── index.html                         ← Mock 사이트 (Premium Learning RPG, 글라스모피즘)
```

**v3가 현행 기준 문서**입니다. v1·v2는 진화 경로의 기록.

---

## 핵심 컨셉

### 차별화 3가지
1. **PoE 보상** — 단순 클릭/체류가 아닌 "본문 80% + 30초 체류" 또는 "능동 응답"으로만 P 적립.
2. **광고주 = 결제처 일치** — 학원이 직접 발행한 수강료 쿠폰이 보상 풀. 광고 단가가 아닌 **CAC**로 직결.
3. **Read-to-Earn + Write-to-Earn 동일 메커니즘** — 사용자 작성 글이 PoE-정독될 때마다 작성자도 P 적립.

### 검증할 핵심 가설 (MVP)
> "분당·판교 학부모는, 신뢰할 수 있는 학원·입시 정보에 본인이 다닐 가능성이 있는 학원의 수강료 쿠폰이 따라온다면, 맘카페보다 이 플랫폼에서 더 깊이 정보를 소비하고 더 자주 돌아온다."

## 도메인 용어

| 용어 | 의미 |
|------|------|
| **PoE** | Proof-of-Engagement — 정독 증명 측정 메커니즘 |
| **P** | 사용자 적립 단위 (Point/Coin). MVP에선 학원 쿠폰 응모권으로만 사용, 환전은 Phase 2 데이터 기반 결정 |
| **B 트랙 (학원)** | 학원이 사전에 쿠폰을 보상 풀에 적립 → 정독자에게 응모권 추첨 |
| **D 트랙 (학원)** | 상담 예약 발생 시 건당 비용 → 정독자에게 일회성 보너스 (4~5개월차 도입) |
| **C 트랙 (작성자)** | 일반 사용자 글이 정독될 때마다 작성자에게 사후 적립 |
| **D 트랙 (작성자)** | 시드 작성자 50명 — 별도 마케팅 예산, 월 고정 페이 + 인센티브 |
| **신뢰도 계수** | 0.3 (신규) → 1.0 (14일 정상) → 1.5 (누적 우수). 보상에 곱해짐 |
| **Loot / 보물상자** | Mock 사이트 UI 용어 — 학원별 쿠폰 풀의 게임적 표현 |

## MVP 범위 (6개월)

**포함**: 반응형 웹앱 / 분당·판교 한정 / 3개 카테고리(학원 후기·입시·교육 일반) / 시드 50명 + 베타 1,000명 / PoE B+D / 학원 30개 파일럿(B 트랙) / 작성자 사후 보상(C) / 휴대폰 본인인증

**제외 (Phase 2+)**: 영상·라이브 / 현금 환전 / 학원 D 트랙 (4~5개월차 도입) / 다른 지역 / 네이티브 앱 / ML 어뷰징 탐지 / Web3

**예산**: 6.7억 원 (인건비 4.5억 + 시드 작성자 9천만 + 기타 1.3억)

## Mock 사이트 디자인 시스템

위치: `mock/index.html` (단일 HTML SPA)

- **컨셉**: Premium Learning RPG Dashboard — 정독=퀘스트, P=코인, 신뢰도=캐릭터 레벨, 응모권=인벤토리, 학원 쿠폰=보물상자
- **스타일**: 글라스모피즘(`backdrop-filter: blur(24px)`) + 3D hover 틸트 + 미드나잇 블루 베이스 + 시안/골드/블루/마젠타 그라디언트 메시
- **폰트**: Bricolage Grotesque(영문 디스플레이) + IBM Plex Sans KR(본문) + Newsreader Italic(숫자 액센트)
- **금지**: Inter/Pretendard/Roboto, 보라 그라디언트 (AI 슬롭 회피)
- **4개 화면**: 홈(피드+미션) / 글 상세(PoE progress) / 인벤토리(지갑+보물상자) / 캐릭터 시트(스탯)

PoE 데모 작동: 글 상세에서 30초+ 체류 + 80% 스크롤 → 토스트 + 잔액 +75 카운터 애니메이션.

## 작업 컨벤션

- **언어**: 한국어 (사용자·기획서·콘텐츠 모두 한국어)
- **문서 작성**:
  - 새 기획 버전은 v숫자 명시, 기존 버전은 보존(아카이브)
  - docx 변환은 PowerShell + Word COM 사용 (Node/Python 미설치 환경)
  - 임시 파일(`_temp_*.html` 등)은 변환 후 즉시 삭제
- **Mock 작업**:
  - 사용자 환경에 Node·npm 없음 → vanilla HTML/CSS/JS, 단일 파일 SPA 선호
  - Google Fonts 외부 import 사용 가능
- **GitHub 푸시**: 글로벌 git config(user.name/email) 비어 있음. push는 사용자가 직접 실행해야 인증 처리 가능
