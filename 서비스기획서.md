# 보상기반 소셜미디어 서비스 기획서

## 1. 프로젝트 개요

### 1.1 서비스명
(가칭) **RewardSocial** — 활동에 대한 정당한 보상을 받는 소셜미디어 플랫폼

### 1.2 한 줄 정의
사용자의 콘텐츠 생산, 큐레이션, 참여 활동을 정량적으로 측정하여 포인트/토큰으로 환산해주는 차세대 소셜미디어 서비스

### 1.3 기획 배경
- 기존 소셜미디어(Instagram, X, TikTok 등)는 사용자가 만든 콘텐츠와 데이터로 막대한 광고 수익을 창출하지만, 정작 가치를 만들어내는 사용자에게는 보상이 돌아가지 않음
- 크리에이터 이코노미(Creator Economy) 시장 성장 — 2026년 기준 글로벌 시장 규모 약 5,280억 달러 추정
- Web3 기반 SocialFi 트렌드와 결합한 새로운 소셜 패러다임에 대한 수요 증가
- MZ세대를 중심으로 "내가 쓴 시간과 콘텐츠에 대한 정당한 대가"에 대한 인식 확산

### 1.4 비전
> **"당신의 시간과 창의성에 정당한 가치를 돌려드립니다."**

## 2. 시장 분석

### 2.1 타겟 시장
| 구분 | 1차 타겟 | 2차 타겟 |
|------|---------|---------|
| 연령 | 20~35세 | 36~45세 |
| 특성 | 디지털 네이티브, 크리에이터 지망생, 부수입 관심층 | 인플루언서, 마케터, 자영업자 |
| 규모 | 국내 약 1,200만 명 | 국내 약 800만 명 |

### 2.2 경쟁사 분석
| 서비스 | 강점 | 약점 | 보상 체계 |
|-------|------|------|----------|
| Instagram | 글로벌 사용자 풀, 비주얼 강점 | 보상 부재, 알고리즘 불투명 | 일부 크리에이터 펀드만 운영 |
| X (Twitter) | 실시간성, 담론 형성 | 광고 수익 분배 제한적 | 프리미엄 구독자 한정 |
| TikTok | 짧은 영상, 추천 알고리즘 | 보상 진입장벽 높음 | 크리에이터 펀드(미국/EU 한정) |
| Steemit / Hive | 블록체인 기반 보상 | UX 열악, 진입장벽 높음 | STEEM/HIVE 토큰 |
| Lens Protocol | 탈중앙화 SNS | 일반 사용자 접근성 낮음 | NFT/소셜 그래프 |

### 2.3 차별화 포인트
1. **모든 사용자에게 보상**: 크리에이터뿐만 아니라 큐레이터, 참여자에게도 보상
2. **투명한 알고리즘**: 보상 산정 기준 공개
3. **법정 화폐 환전 가능**: 포인트 → 현금 환전 지원 (Web3 토큰 옵션도 별도 제공)
4. **광고 수익 분배 모델**: 광고 노출/클릭 수익의 일정 비율을 관련 사용자에게 환원

## 3. 핵심 기능 정의

### 3.1 사용자 활동별 보상 체계

#### 콘텐츠 생산 보상
- **포스트 작성**: 기본 포인트 + 품질 점수 기반 추가 포인트
- **사진/영상 업로드**: 미디어 형식별 차등 보상
- **장문 글/뉴스레터**: 분량/완성도 기반 보상
- **라이브 스트리밍**: 시청 시간 기반 실시간 적립

#### 큐레이션 보상
- **좋아요/리액션**: 트렌드 발굴 시 추가 보상 (early curator bonus)
- **댓글 작성**: 양질의 댓글에 대한 커뮤니티 평가 기반 보상
- **공유/리포스트**: 확산 기여도 기반 보상
- **북마크/컬렉션**: 큐레이션 활동 보상

#### 참여 활동 보상
- **일일 출석**: 연속 출석 보너스 (스트릭 시스템)
- **친구 초대**: 초대받은 사용자 활동의 일정 비율
- **신고/모더레이션**: 정확한 신고에 대한 보상
- **설문/이벤트 참여**: 활동별 차등 지급

### 3.2 보상 산정 알고리즘 (개념도)

```
사용자 보상 = (기본 활동 포인트)
           × (콘텐츠 품질 계수: 0.5 ~ 3.0)
           × (커뮤니티 반응 계수: 0.8 ~ 5.0)
           × (사용자 신뢰도 계수: 0.5 ~ 2.0)
           - (어뷰징 페널티)
```

### 3.3 주요 화면/기능 명세

#### (1) 피드 (Feed)
- 추천/팔로잉/트렌딩 탭
- 게시물별 예상 보상 표시 (작성자/큐레이터)
- 광고 게시물 명확 구분 (광고 수익 분배 정보 표시)

#### (2) 작성 (Compose)
- 텍스트/이미지/영상/라이브 통합 에디터
- 콘텐츠 작성 전 예상 보상 시뮬레이션
- 해시태그/카테고리 기반 보너스 풀 안내

#### (3) 지갑 (Wallet)
- 누적/일별/월별 보상 내역
- 포인트 → 현금/기프티콘/토큰 환전
- 환전 수수료 및 최소 환전 금액 안내
- 세금 자료 자동 생성 (3.3% 원천징수 등)

#### (4) 마이페이지
- 신뢰도 점수 (Trust Score) 표시
- 활동 통계 대시보드
- 콘텐츠 성과 분석 리포트

#### (5) 마켓플레이스 (선택)
- 크리에이터 굿즈/디지털 콘텐츠 판매
- 보유 포인트로 직접 결제 가능

## 4. 비즈니스 모델

### 4.1 수익 구조
| 수익원 | 비중(예상) | 설명 |
|--------|-----------|------|
| 광고 수익 | 60% | 네이티브 광고, 디스플레이 광고 (수익의 30~50%는 사용자에게 환원) |
| 프리미엄 구독 | 20% | 광고 제거, 추가 보상 부스트, 분석 도구 등 |
| 환전 수수료 | 10% | 포인트 → 현금 환전 시 5~10% |
| 마켓플레이스 수수료 | 7% | 크리에이터 콘텐츠 거래 수수료 |
| 기업 파트너십 | 3% | 브랜드 캠페인, B2B API |

### 4.2 보상 풀 운영 방식
- 광고 수익의 **40%**를 보상 풀로 적립
- 일/주/월 단위로 활동 점수 비례 분배
- 신규 가입자 유치를 위한 별도 그로스 풀 운영 (초기 6개월 한정)

## 5. 기술 아키텍처 (개요)

### 5.1 기술 스택 (안)
- **프론트엔드**: React Native (모바일), Next.js (웹)
- **백엔드**: Node.js + NestJS, Python (ML/추천)
- **데이터베이스**: PostgreSQL (트랜잭션), MongoDB (콘텐츠), Redis (캐시)
- **검색/추천**: Elasticsearch, ML 기반 개인화 추천
- **미디어 처리**: AWS S3, CloudFront, MediaConvert
- **결제/환전**: PG사 연동(토스/PortOne), 가상자산 사업자 연계(선택)

### 5.2 보안/어뷰징 방지
- 디바이스 핑거프린팅 + 행동 패턴 분석
- AI 기반 이상 활동 탐지 (어뷰징/봇 계정 차단)
- 휴대폰 본인인증 (1인 1계정 원칙)
- 콘텐츠 검열 시스템 (텍스트/이미지 AI 모더레이션)

## 7. 로드맵

### Phase 1: MVP (0~6개월)
- 핵심 기능: 회원가입, 피드, 게시물 작성, 좋아요/댓글, 기본 보상 시스템
- 베타 클로즈드 테스트 (1,000명)
- 포인트 시스템만 운영 (환전 X)

### Phase 2: 정식 출시 (6~12개월)
- 환전 기능 오픈 (현금/기프티콘)
- 라이브 스트리밍, 영상 업로드 추가
- 광고 시스템 도입
- 목표: MAU 50만

### Phase 3: 확장 (12~24개월)
- 크리에이터 마켓플레이스
- 프리미엄 구독 출시
- 토큰/Web3 옵션 도입 검토
- 글로벌 진출 (영어/일본어/베트남어)
- 목표: MAU 300만

## 11. 결론

본 서비스는 **사용자 가치 환원**이라는 명확한 차별점을 통해, 포화된 소셜미디어 시장에서 새로운 카테고리를 개척할 수 있는 잠재력을 가지고 있습니다. 다만 **어뷰징 방지**, **보상 풀의 지속가능성**, **법적 규제 대응**이 성공의 핵심 변수이며, 단계적이고 신중한 접근이 필요합니다.
