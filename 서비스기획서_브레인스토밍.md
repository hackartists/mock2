# 보상기반 소셜미디어 서비스 기획서 (Brainstorming Edition)

> 본 문서는 발산적 사고(Divergent) → 다관점 탐색(Multi-Perspective) → 비판적 검증(Critique) → 수렴(Convergence)의 4단계 브레인스토밍 워크플로우로 작성되었습니다.

---

## STEP 0. 문제 재정의 (Problem Reframing)

### 표면 문제
"보상기반 소셜미디어를 만들고 싶다"

### 5 Whys
1. **왜** 보상기반인가? → 사용자가 만든 가치를 가져가는 플랫폼이 정당하지 않다고 느껴서
2. **왜** 정당하지 않은가? → 콘텐츠를 생산해도 광고주와 플랫폼만 돈을 번다
3. **왜** 그런 구조가 유지되는가? → 사용자에게 분배할 측정/지급 인프라가 없었고, 광고 비즈니스가 이미 굳어졌기 때문
4. **왜** 지금이 기회인가? → Web3, Stripe Connect, 토스 비즈니스 등으로 마이크로 페이먼트가 가능해짐
5. **왜** 사용자가 옮겨올 것인가? → 기존 SNS에 피로감 + 부수입에 대한 욕구 + Z세대의 "정당한 보상" 가치관

### 진짜 풀어야 할 문제
**"수많은 사람이 이미 만들어내고 있는 디지털 가치를, 어떻게 정확히 측정하고 지속가능하게 분배할 것인가?"**

### 풀지 않을 문제 (Out of Scope)
- 블록체인 자체의 기술적 진보
- 글로벌 즉시 진출
- 모든 콘텐츠 포맷 동시 지원
- 기존 SNS 기능 100% 복제

## STEP 1. 발산 (Divergent Thinking)

### 1.1 Crazy 8s — 극단적 컨셉 8개

| # | 컨셉 | 한 줄 설명 | 차별화 |
|---|------|-----------|--------|
| 1 | **타임토큰 (TimeToken)** | SNS에서 보낸 시간만큼 토큰 적립 | "광고 보는 시간 = 내 돈" |
| 2 | **큐레이터 우선 (CuratorFirst)** | 글 쓰는 사람보다 발굴하는 사람에게 더 큰 보상 | 트렌드 헌터 경제 |
| 3 | **앤티(Anti)-알고리즘** | 추천 알고리즘 없음, 사용자가 직접 카테고리 등록·구독·보상 | 투명성 극대화 |
| 4 | **펀딩형 SNS** | 모든 게시물이 후원·구독 가능한 미니 펀딩 | 1포스트 = 1프로젝트 |
| 5 | **Proof-of-Engagement** | 단순 좋아요 X, 시청시간/댓글 깊이/공유경로로만 보상 | 어뷰징 원천 차단 |
| 6 | **로컬 SNS** | 동네/지역 단위로 묶어서 보상 풀 운영 | 당근마켓 + 트위터 |
| 7 | **DAO 운영** | 보상 정책을 사용자 투표로 결정하는 거버넌스 SNS | 진짜 사용자 소유 |
| 8 | **AI 협업 SNS** | 사용자 + AI가 공동 창작, 보상도 사용자 70% / AI 풀 30% | 2026년형 크리에이터 |

### 1.2 SCAMPER — 기존 SNS를 변형하면

- **S**ubstitute: 광고 노출 → 후원 노출로 대체
- **C**ombine: 트위터 + 토스페이먼츠 + 슬랙 워크스페이스
- **A**dapt: 게임의 시즌패스 시스템을 SNS에 이식
- **M**odify: 팔로워 수 → "보상 분배 네트워크 크기"로 재정의
- **P**ut to other use: 댓글창을 마이크로 토론장 + 베팅장으로
- **E**liminate: 광고 자체를 없애고 구독+후원만으로
- **R**everse: 인플루언서가 팔로워에게 돈을 주는 구조 (역방향 어텐션 경제)

### 1.3 와일드 아이디어 (검열 없이)
- 게시물 작성 시 "이 글이 N번 공유되면 자동으로 출금되는" 스마트 컨디션
- 익명 모드 게시물도 사후 신원 공개 시 누적 보상 회수 가능
- "오늘 안 쓰면 보상 사라짐" 형태의 의도된 FOMO
- 콘텐츠를 NFT로 박제하지 않고, 영향력 점수만 박제
- 좋아요 = 0.1원 직접 송금 (마이크로 후원의 극단형)
- 모든 사용자가 자기만의 "광고 단가"를 설정 가능
- 신뢰 점수가 낮은 계정의 댓글은 가시성 자동 축소
- 독서/시청에 보상하는 "Read-to-Earn" 모드 (콘텐츠 끝까지 보면 0.x원)

## STEP 2. 다관점 탐색 (Multi-Perspective)

### 2.1 4명의 가상 페르소나 인터뷰

#### 👩 **지혜 (24, 대학생, 헤비 SNS 유저)**
> "솔직히 인스타에 매일 2시간씩 쓰는데, 한 푼도 안 받는 게 억울했어요. 근데 보상받자고 가입했다가 1만 원 빼려고 한 달 걸리면 그것도 짜증날 듯. **최소 환전 기준이 낮아야** 해요. 5천 원이면 좋겠어요."

→ **인사이트**: 환전 마찰(friction)이 핵심 UX 지표

#### 👨 **민준 (32, 마케터, 인플루언서)**
> "팔로워 3만인데, 협찬 받으려면 에이전시 거쳐야 하고 수수료 떼이고 정산 두 달 걸려요. **광고주와 직접 매칭**되고, **즉시 정산**되는 게 최고예요. 근데 어뷰징 봇 많으면 광고주가 안 들어와요."

→ **인사이트**: B2B(광고주) 신뢰가 보상 풀의 지속가능성을 결정

#### 🧓 **정훈 (45, 자영업 사장, 라이트 유저)**
> "동네 가게인데 인스타 광고는 너무 비싸요. 동네 사람들이 보고 보상도 받는 SNS면 광고비를 흔쾌히 낼 텐데. **로컬 타겟팅**이 진짜 중요해요."

→ **인사이트**: 로컬/니치 마켓이 초기 침투 포인트일 수 있음

#### 🧑‍💻 **하늘 (28, 개발자, 비판적 유저)**
> "토큰이코노미라는 말 들으면 일단 도망가요. 폰지 같아 보여서. **현금만** 받고 싶어요. 그리고 보상 산정 기준이 블랙박스면 X스팀잇 됩니다. **공식 + 코드 공개**해야 해요."

→ **인사이트**: 투명성과 "토큰 알레르기"가 진입장벽

### 2.2 적대적 페르소나 (Devil's Advocate)
- **벤처캐피탈리스트**: "보상 풀은 광고 수익에 의존하는데, 광고가 안 들어오면? 폰지 구조 아닌가?"
- **법무 자문**: "포인트 환전은 선불전자지급수단 발행업 등록 대상일 수 있고, 개당 단가가 작아도 누적되면 자금세탁 모니터링 의무 발생합니다."
- **세무사**: "사용자에게 지급되는 보상은 기타소득(8.8%) 또는 사업소득(3.3%) 원천징수 대상. 연 300만원 초과 시 종합소득 신고 안내 의무."
- **기존 인플루언서**: "지금 광고 단가가 더 높은데 굳이 옮겨갈 이유?"

## STEP 3. 비판적 검증 (Critique)

### 3.1 구조적 약점 5가지

| # | 약점 | 본질 | 대응 가설 |
|---|------|------|----------|
| 1 | **콜드 스타트** | 사용자 없으면 보상 풀 없고, 보상 풀 없으면 사용자 없음 | 초기 6개월 그로스 풀(투자금) + 인플루언서 시드 그룹 |
| 2 | **어뷰징** | 1원이라도 보상이 있으면 봇이 들어옴 | KYC + 행동 패턴 ML 탐지 + 신뢰도 점수 |
| 3 | **법적 리스크** | 환전 = 금융업, 토큰 = VASP 이슈 | MVP는 "기프티콘 교환" 모드로 시작 → 점진적 현금 전환 |
| 4 | **단가 압력** | 사용자당 ARPU(매출)가 보상으로 빠지면 회사가 적자 | 광고 외 수익원(구독/마켓플레이스) 비중 확대 |
| 5 | **관성 (Stickiness)** | "보상 때문에 옮겨온 사용자"는 보상이 줄면 떠남 | 커뮤니티/관계 기반 록인 (보상은 +α) |

### 3.2 Pre-mortem — "2년 후 망했다면, 왜?"
1. 보상 풀 고갈로 보상이 줄자 헤비유저가 한꺼번에 이탈
2. 어뷰징 봇이 보상의 30%를 가져가버려 정직한 사용자에게 돌아갈 몫이 줄어듦
3. 금융감독원이 "선불전자지급수단" 미등록으로 시정 명령
4. 진짜 크리에이터들은 결국 유튜브/인스타로 돌아감 (도달률이 더 좋음)
5. "용돈벌이 앱" 이미지가 굳어져 광고주가 외면

### 3.3 망하지 않으려면 — 대응 원칙
1. **"보상은 동기부여, 록인은 관계"** — 커뮤니티 기능을 보상보다 먼저 강화
2. **"먼저 정산하고 나중에 운다"** — 어뷰징 의심돼도 작은 금액은 일단 지급, ML로 패턴 학습
3. **법적 준비를 MVP보다 먼저** — 변호사 자문 → 사업 모델 → 개발 순서
4. **광고 외 수익원을 1년 내 확보** — 구독, 마켓, B2B API 동시 추진
5. **헤비유저 이탈을 KPI에 포함** — 단순 MAU가 아니라 "30일 잔존율 × 활동 강도"

## STEP 4. 수렴 (Convergence)

### 4.1 핵심 컨셉 결정

**최종 컨셉: "Proof-of-Engagement 기반 로컬-니치 SNS"**

- Crazy 8s 중 **#5(Proof-of-Engagement)** + **#6(로컬 SNS)** 결합
- 이유:
  - PoE는 어뷰징(가장 큰 리스크) 대응에 가장 강함
  - 로컬/니치는 콜드 스타트 문제를 좁은 시장에서 해결 가능
  - 두 컨셉 모두 "투명성"과 "지속가능성"이라는 비판적 검증의 답

### 4.2 한 문장 정의
> **"내가 진심으로 시간을 쓴 만큼, 내 동네/관심사 안에서 정산되는 소셜미디어"**

### 4.3 핵심 차별화 3가지
1. **시간 가중치 보상**: 좋아요 1회 ≠ 30초 정독 후 좋아요 1회
2. **카테고리(또는 지역) 단위 보상 풀**: 전체 풀이 아닌 카테고리별 풀로, 작은 커뮤니티도 의미 있는 보상
3. **현금 우선, 토큰 옵션**: 기본은 원화 환전, Web3 옵션은 별도 모드

### 4.4 MVP 범위 — "최소 검증 가능한 가설"
**검증할 가설**: *"30초 이상 정독에 0.x원의 즉시 보상이 있으면, 사용자는 더 깊이 콘텐츠를 소비한다"*

| 포함 | 제외 |
|------|------|
| 텍스트 + 이미지 게시물 | 영상, 라이브 |
| 1개 카테고리(예: 재테크) 한정 | 전체 카테고리 |
| Read-to-Earn (시청 시간 보상) | 작성/큐레이션 보상 (Phase 2) |
| 기프티콘 교환만 | 현금 환전 (법무 검토 후) |
| 웹앱 (반응형) | 네이티브 모바일 앱 |
| 1,000명 클로즈드 베타 | 일반 공개 |

### 4.5 6개월 마일스톤

| 주차 | 목표 | 핵심 산출물 |
|------|------|------------|
| W1-4 | 법무 자문 + 시장 검증 | 변호사 의견서, 랜딩페이지 사전등록 500명 |
| W5-12 | MVP 개발 (단일 카테고리) | 웹앱, 보상 엔진, 어뷰징 탐지 v1 |
| W13-16 | 클로즈드 베타 1,000명 | DAU, 30일 잔존율, 평균 정독 시간 |
| W17-20 | 가설 검증 + 피벗 결정 | 데이터 리포트, Phase 2 의사결정 |
| W21-24 | Phase 2 준비 (정식 출시) | 환전 인프라, 광고주 파트너 5곳 |

### 4.6 성공 지표 (Leading vs Lagging)

**Leading 지표 (선행)**
- 평균 정독 시간 (목표: 45초+)
- 첫 보상 수령까지 걸린 시간 (목표: 가입 후 24시간 이내)
- 30일 잔존율 (목표: 40%+)

**Lagging 지표 (후행)**
- MAU (6개월 1만, 1년 20만)
- ARPU (1년 차 3,000원)
- 광고주 재계약률 (목표: 60%+)
